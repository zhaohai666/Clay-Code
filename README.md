# 1. 项目概述
## 1.1 产品定位
ClayCode 是**完全从零原生重写**的一体化终端AI编码工具，无 OpenCLI / OpenClaw 等外部重型Agent框架依赖。
底层拆分为可独立维护的内部NPM分包，内置两大自研核心能力：
1. 浏览器自动化桥接模块：复用本机Chrome登录Cookie会话，对接豆包、ChatGPT、Claude网页版，无需API Key、零推理成本；
2. 本地操作系统执行引擎：原生实现文件增量Diff编辑、目录扫描、Shell执行、Git工程操作，支持AI多轮自动迭代修复代码。

主程序单进程运行，模块间内存对象直传，消除跨进程、HTTP、外部子进程带来的兼容损耗；同时通过分包、插件层预留完整扩展能力，兼顾性能与长期迭代。

## 1.2 核心设计目标
1. **底层零重型第三方依赖**：根除多工具联动产生的PATH、沙箱、浏览器锁、输出流污染等兼容问题；
2. **免密钥低成本推理**：复用网页AI免费对话额度，无模型接口付费成本；
3. **原生工程闭环能力**：AI读取本地项目、执行编译脚本、捕获报错、自动生成修复代码；
4. **分层模块化可扩展**：核心能力拆分子包，插件系统支持自定义适配器/工具集；
5. **全链路安全可控**：文件/命令白名单、敏感会话加密、高风险操作沙箱隔离；
6. **极简运维+可观测**：单包一键部署、分级日志、性能监控指标、自动化测试覆盖；
7. **跨平台统一体验**：macOS/Windows/Linux一套底层处理编码、换行、路径逻辑。

## 1.3 能力横向对标
| 能力项 | ClayCode（原生自研优化版） | Claude Code（官方API方案） | OpenCLI+OpenClaw粘合方案 |
|--------|---------------------------|---------------------------|-------------------------|
| AI调用方式 | 浏览器网页会话，免API Key；预留Ollama本地模型入口 | 付费官方API，必须密钥充值 | 双第三方工具粘合，多进程隔离 |
| 文件增量编辑 | 自研Diff补丁引擎，独立分包维护 | 官方内置 | 依赖OpenClaw文件工具 |
| Shell/Git执行 | 原生封装+命令白名单安全校验 | 官方内置无严格沙箱 | 依赖OpenClaw执行能力，无内置安全拦截 |
| 进程模型 | 单进程内核，分包模块化，内存直传数据 | 独立二进制客户端 | 至少4套隔离进程，多跨进程通信损耗 |
| 浏览器管理 | 内置单例生命周期管控，缓存锁自动清理 | 不支持网页会话复用 | 多进程易产生Chrome锁、端口冲突 |
| 扩展能力 | 插件系统、独立分包、开放SDK、自定义适配器 | 仅官方提供有限配置 | 第三方更新极易破坏整合链路，无标准化扩展接口 |
| 安全机制 | 命令白名单、目录越权拦截、AES会话加密、可选轻量沙箱 | 基础权限控制，无本地数据加密 | 无统一安全隔离层，依赖外部工具策略 |

| 可观测运维 | 分级滚动日志、Prometheus监控指标、完整自动化测试 | 基础日志输出，无监控指标 | 日志分散多程序，无统一监控体系 |

# 2. 整体架构设计
## 2.1 分层分包模块化架构总览
整体采用「内核主程序 + 独立可分包核心能力 + 插件扩展层」三层架构，所有模块运行于同一Node.js主进程，无外部独立服务：
```
┌─────────────────────────────────────────────────────┐
│                  终端交互入口层                      │
│  命令解析、配置加载、终端UI渲染、命令补全(oclif)      │
└───────────────────────┬─────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                   插件扩展层（预留）                 │
│  自定义AI适配器插件 / 自定义本地工具插件 / 事件钩子    │
└───────────────────────┬─────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                 核心能力分包层（独立NPM包）          │
│  ├─ @claycode/browser-bridge    网页AI浏览器桥接     │
│  ├─ @claycode/protocol-convert  协议转换&上下文管理  │
│  └─ @claycode/local-executor    本地文件/命令执行引擎│
└───────────────────────┬─────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│                  底层公共基础层                      │
│  加密工具、缓存管理器、日志组件、错误统一处理、工具SDK│
└─────────────────────────────────────────────────────┘
```

分包优势：
1. 各核心能力独立单元测试、独立版本迭代；
2. 支持按需替换/重写单一分包，不改动整体内核流程；
3. 对外可单独导出分包SDK，供第三方程序调用。

## 2.2 全链路数据流时序图
1. 用户终端执行 `clay agent "重构项目SQL代码"`
2. 终端交互模块解析参数，加载全局加密配置、项目路径信息
3. 上下文管理器读取项目文件摘要（L1内存缓存优先），组装完整TaskRequest对象
4. 桥接模块复用常驻Chrome单例，流式发起网页AI请求，内存清洗输出文本
5. 协议转换模块解析自由文本，标准化生成ToolCall指令数组
6. 本地执行引擎按白名单校验文件/命令合法性，增量修改文件、执行Shell
7. 捕获执行输出、报错、文件Diff，生成ExecuteResult结果对象
8. 上下文管理器将执行结果增量同步至AI对话上下文，自动发起下一轮推理
9. 任务完成后持久化会话加密存储，释放非活跃内存缓存，终端输出执行总结

## 2.3 插件扩展层设计规范
### 2.3.1 插件分类
1. AI适配器插件：新增Kimi、通义千问等网页模型适配；
2. 本地工具插件：扩展Docker、Redis CLI、K8s等自定义执行指令；
3. 事件钩子插件：会话创建/任务结束/浏览器重启等生命周期钩子。
### 2.3.2 标准插件API
统一导出固定生命周期函数：`setup()`、`beforeAiRequest()`、`afterToolExecute()`、`destroy()`；
插件目录统一规范，主程序自动扫描 `~/.claycode/plugins` 目录加载，支持热重载。

# 3. 核心模块技术实现
## 3.1 终端交互解析模块
### 3.1.1 底层框架
基于 `oclif` 实现标准化命令体系，原生支持终端命令自动补全、分层帮助文档、子命令嵌套。
### 3.1.2 完整支持命令清单
```bash
# 批量工程编码任务，自动多轮迭代
clay agent "需求描述"
# 交互式持续对话会话
clay chat
# 唤起内置浏览器登录网页AI，持久加密保存Cookie
clay login
# 全局配置管理
clay config set [key] [value]
clay config list
# 会话管理：新建/切换/恢复/删除
clay session new [session-name]
clay session use [session-name]
clay session recover
# 日志查看、日志分级过滤
clay log --level error
# 插件管理
clay plugin install xxx
clay plugin list
```
### 3.1.3 全局配置持久化
存储路径：`~/.claycode/config.json`，敏感字段（浏览器Cookie密钥、加密密钥）不明文存储，采用AES加密落地。
```json
{
  "defaultAdapter": "doubao",
  "browserDataPath": "~/.claycode/browser-cache",
  "maxContextChunk": 12000,
  "requestTimeout": 90000,
  "autoSyncExecuteLog": true,
  "browserHeadless": false,
  "execWhiteList": ["git", "npm", "mvn", "pnpm"],
  "enableSandbox": false,
  "cacheLevel": "l1+l2"
}
```

## 3.2 `@claycode/browser-bridge` 网页AI桥接模块
完全替代OpenCLI全套能力，内置Puppeteer浏览器内核，独立分包可单独测试发布：
1. **全局单例浏览器生命周期管控**
   进程启动自动扫描并删除 `.lock`、`.incomplete` 缓存锁文件，全局唯一Chrome实例；独立缓存目录，与本机用户Chrome隔离，杜绝端口、缓存冲突。
2. **本机Chrome会话复用**
   读取本地Chrome用户目录Cookie，`clay login` 手动唤起浏览器完成人机验证，会话数据AES加密持久化。
3. **多AI适配器抽象层**
   统一抽象输入框、发送按钮、回复内容选择器，新增模型仅需开发独立适配器插件，不改动主流程。
4. **内存级流式输出清洗**
   AI网页返回流在内存过滤ANSI色彩码、进度条、广告杂讯，仅保留代码与指令文本，无文件中转损耗。
5. **超时、重试、异常上报**
   单轮请求超时90s，指数退避重试3次；检测人机验证弹窗返回标准化错误码，终端输出修复指引。

## 3.3 `@claycode/protocol-convert` 协议转换与上下文管理器
系统调度核心，独立分包，负责文本与工具指令互转、上下文增量同步：
1. **智能ToolCall解析引擎**
   正则+代码块识别规则，自动区分文件读写、增量Diff编辑、Shell、Git、目录遍历指令，输出标准化结构化对象。
2. **增量上下文同步（性能核心优化）**
   仅将变更文件、最新执行报错片段追加至AI上下文，不重复传输完整项目代码，大幅减少网页传输数据量。
3. **超长上下文分块压缩**
   超过`maxContextChunk`阈值自动切割文件分批传入AI，避免网页对话长度截断。
4. **统一会话内存缓存**
   L1内存常驻当前会话记录，任务结束自动加密落地至L2磁盘缓存；非活跃会话自动释放内存。

## 3.4 `@claycode/local-executor` 原生本地执行引擎
自研无OpenClaw依赖，内置完整安全校验层，独立分包：
### 3.4.1 文件系统工具集
- `readFile`：统一UTF-8编码读取；
- `writeFile`：自动创建多级目录，限制操作路径不超出项目根目录（防越权）；
- `editPatch`：增量Diff补丁局部修改，不全量覆盖文件；
- `glob`：递归遍历目录，自动过滤 `.gitignore` 文件生成项目摘要。
### 3.4.2 命令执行安全体系
1. **命令白名单机制**：仅允许配置内预设指令，未知命令直接拦截并返回友好提示；
2. 跨平台自动适配：Windows cmd / macOS/Linux bash 区分处理；
3. 完整捕获stdout/stderr、退出码，流式实时输出；
4. 可选轻量沙箱（Node vm / Docker容器）隔离高风险未知命令。
### 3.4.3 Git内置工具
原生封装安全Git指令，变更记录自动同步至AI上下文。

# 4. 数据协议与通信规范
模块间内存直接传递JS对象，JSON序列化仅用于磁盘持久化；配套完整JSON Schema用于参数校验。
## 4.1 核心TS类型定义
```typescript
interface TaskRequest {
  userPrompt: string;
  projectPath: string;
  projectChunkContext: string;
  sessionHistory: ChatMessage[];
  lastExecuteOutput: ExecuteResult;
  config: GlobalConfig;
}

interface ChatMessage {
  role: "user" | "ai" | "system";
  content: string;
}

interface ExecuteResult {
  success: boolean;
  outputText: string;
  errorText: string;
  diffPatch?: string;
  operateFiles: string[];
}

interface TaskResponse {
  code: number;
  msg: string;
  data: {
    toolCalls: ToolCall[];
    appendChatMsg: ChatMessage;
  }
}

type ToolType = "read" | "edit" | "write" | "bash" | "git" | "glob";
interface ToolCall {
  tool: ToolType;
  filePath?: string;
  patch?: string;
  command?: string;
  globPattern?: string;
}
```
## 4.2 ToolCall JSON Schema
用于入参自动校验，拦截非法文件路径、危险命令：
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "tool": {"type": "string", "enum": ["read","edit","write","bash","git","glob"]},
    "filePath": {"type": "string"},
    "patch": {"type": "string"},
    "command": {"type": "string"},
    "globPattern": {"type": "string"}
  },
  "required": ["tool"]
}
```
## 4.3 全局统一错误码体系
| 错误码 | 类型说明 | 自动处理策略 |
|--------|---------|-------------|
| 0 | 正常成功 | 继续执行工具调用 |
| 1001 | 浏览器锁文件冲突 | 自动清理缓存、重启Chrome实例 |
| 1002 | 网页AI需要人机验证 | 终端引导执行 `clay login` |
| 1003 | AI请求超时 | 指数退避重试3次，超限终止任务 |
| 1004 | 上下文超长截断 | 自动拆分项目分块重新请求AI |
| 2001 | 文件越权/权限不足 | 报错同步AI，生成权限修复方案 |
| 2002 | Shell命令不在白名单 | 终端提示配置execWhiteList |
| 2003 | 命令执行异常 | 日志同步AI，自动迭代修复代码 |
| 3001 | 适配器不存在 | 提示切换合法AI适配器 |
| 4001 | 会话数据解密失败 | 引导重新登录同步会话 |

# 5. 性能优化与资源管理
## 5.1 分级内存缓存策略
- L1缓存（进程内存）：高频访问文件内容、当前会话完整对话；会话闲置5分钟自动回收释放；
- L2缓存（本地磁盘）：项目摘要、历史会话快照，持久化加密存储；
- 缓存淘汰规则：LRU淘汰，限制最大内存占用阈值。
## 5.2 增量上下文同步机制
仅对比上一轮文件变更列表，只将修改片段、执行报错追加至AI对话，避免全量重复传输上万行项目代码，大幅降低网页AI交互耗时。
## 5.3 流式并行处理方案
1. 文件读取与AI网络请求并行执行 `Promise.allSettled`；
2. AI流式响应边解析边执行工具调用，无需等待完整回复；
3. 大文件采用Node Stream分片读写，避免一次性加载超大文本占用内存。
## 5.4 资源自动回收策略
1. 闲置会话5分钟释放L1内存缓存；
2. 浏览器闲置10分钟自动释放，保留加密Cookie缓存；
3. 任务结束清空临时变量、销毁临时文件句柄，防止内存泄漏。

# 6. 安全与容错机制
## 6.1 Shell/文件操作白名单隔离
1. Shell命令仅放行配置内预设指令，未知命令直接拦截；
2. 文件操作强制校验路径前缀，禁止访问项目目录以外系统文件（/etc、C:\Windows等）；
3. 批量文件操作增加文件数量上限，防止恶意批量遍历系统目录。
## 6.2 本地敏感数据AES加密存储
浏览器Cookie、会话对话、登录凭证采用AES-256-CBC加密落地；配置文件仅存储加密密钥索引，不保存明文隐私数据。
## 6.3 轻量沙箱风险操作隔离
可选开启 `enableSandbox` 配置：
1. 简单场景使用Node.js vm隔离代码片段；
2. 高危Shell命令支持Docker临时容器执行，隔离主机文件系统。
## 6.4 全链路异常自动恢复策略
1. Chrome进程崩溃：销毁实例、清理锁文件、重启浏览器，保留加密会话；
2. 网络波动：指数退避重试，最大3次；
3. 会话文件损坏：`clay session recover` 读取磁盘备份快照恢复；
4. AI返回无合法ToolCall：自动追加标准化提示词重新发起请求。

# 11. 附录：技术规范与术语表
## 11.1 项目本地目录完整存储规范
```
~/.claycode/
├─ config.json               # 全局配置文件（敏感字段加密）
├─ browser-cache/             # Chrome浏览器Cookie、缓存（AES加密）
├─ sessions/                 # 会话持久化加密JSON文件
│  └─ [session-id].json
├─ logs/                     # 分级滚动日志文件
│  └─ claycode-20260702.log
├─ cache/                    # L2磁盘项目摘要缓存
├─ plugins/                  # 用户自定义插件存放目录
└─ temp/                     # 临时文件、补丁缓存
```
## 11.2 核心专业术语释义
1. 增量上下文同步：仅传输本轮变更代码片段，不重复全量项目；
2. Diff补丁编辑：仅替换指定代码片段，不全量覆盖整个文件；
3. 单例浏览器托管：全局唯一Chrome进程，统一管理生命周期；
4. ToolCall：标准化本地操作指令结构体，统一AI与执行引擎交互格式；
5. 分包模块化：核心能力拆分为独立NPM包，解耦便于单独维护测试。
## 11.3 部署常用命令速查表
```bash
# 环境自检
clay doctor
# 初始化网页AI登录会话
clay login
# 配置放行Shell命令白名单
clay config set execWhiteList '["git","npm","pnpm"]'
# 查看错误日志
clay log --level error
# 安装第三方插件
clay plugin install xxx
# 会话损坏恢复
clay session recover
```

# 优化价值总结
1. **技术深度增强**：新增完整安全隔离、分级缓存、可观测运维、自动化测试体系，底层架构严谨可控；
2. **可扩展性拉满**：分包解耦+标准化插件系统+对外SDK，支持社区二次开发与能力替换；
3. **用户体验升级**：终端可视化、智能报错修复指引、命令补全，降低上手门槛；
4. **长期迭代兼容**：预留本地模型、分布式、Web面板扩展接口，适配中长期技术演进；
5. **安全体系完善**：命令白名单、路径越权拦截、AES会话加密、可选沙箱，规避本地文件/系统操作风险；
6. **性能专项优化**：分级缓存、增量上下文、流式并行处理，大幅降低内存与网络传输开销。

整体全程坚守「原生自研、无第三方重型依赖」核心设计原则，所有新增优化均基于内部模块化重构实现，不引入外部Agent框架、浏览器桥接工具等第三方重型组件。
